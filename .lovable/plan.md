## الهدف
تحويل لوحة الإدارة الحالية إلى مركز واضح لتطوير وإدارة المشاريع الأخرى من هنا، مع احترام قاعدة أساسية: لا حذف ولا تغيير كبير إلا بعد موافقتك الصريحة.

## ما لن نفعله
- لن نغير نظام الدخول الآن، لأن دخول المدير يعمل خارج Preview والمشكلة خاصة ببيئة المعاينة.
- لن نحذف صفحات أو جداول أو بيانات.
- لن نسمح بتسجيل مستخدمين جدد.
- لن ننفذ تعديل جماعي تلقائي على المشاريع الأخرى بدون زر تأكيد واضح.

## ما سنطوره

### 1. تثبيت قاعدة الأمان داخل الواجهة
- إزالة/تعطيل أزرار الحذف النهائي من صفحات الإدارة التي تخص المشاريع والمواقع، أو تحويلها إلى إجراء آمن مثل “تعطيل/أرشفة”.
- أي إجراء خطر يظهر بتحذير واضح قبل التنفيذ.
- المحافظة على صلاحيات المدير الحالية كما هي.

### 2. تحسين مركز التحكم الحالي
- جعل `/admin/control-center` هو المكان الرئيسي لإدارة المواقع الأخرى.
- إضافة حالات واضحة لكل موقع:
  - نشط
  - معطّل مؤقتاً
  - يحتاج مراجعة
  - جاهز لتلقي أوامر
- تحسين عرض الروابط: رابط الموقع، رابط Lovable، رابط GitHub إن وجد.
- إضافة بحث وتصنيف أوضح للمواقع.

### 3. إضافة نظام “أوامر تطوير” للمشاريع الأخرى
عندما تكتب أمراً وتحدد أنه يشمل مشاريع أخرى، يتم تسجيله كأمر داخل اللوحة، وليس تنفيذه عشوائياً.

مثال:
```text
الأمر: عدّل صفحة الأسعار وأضف زر واتساب
النطاق: كل المشاريع / مشاريع محددة فقط
الحالة: مسودة → بانتظار موافقة → جاهز للتنفيذ → تم
```

الميزات:
- إنشاء أمر تطوير جديد.
- اختيار المشاريع المستهدفة.
- تحديد هل يشمل كل المشاريع أو مشاريع محددة فقط.
- حفظ الأمر في سجل واضح.
- عرض حالة كل مشروع بالنسبة للأمر.
- زر “اعتماد الأمر” قبل التنفيذ أو النسخ.

### 4. طريقة التطبيق العملية حسب الإمكانيات الحالية
لأن Lovable لا يوفر API مباشر لتعديل كود مشاريع أخرى من داخل مشروع واحد، سنستخدم طريقة آمنة وواضحة:
- اللوحة تسجل أوامر التطوير لكل مشروع.
- يمكن نسخ الأمر الجاهز لكل مشروع إلى محادثة Lovable الخاصة بذلك المشروع.
- إن كان لديك Webhook أو سيرفر خارجي لاحقاً، يمكن ربطه بدون إعادة بناء اللوحة.

### 5. قاعدة بيانات بسيطة للأوامر
سنضيف جداول جديدة فقط، بدون تعديل أو حذف الجداول الحالية:
- جدول أوامر التطوير.
- جدول يربط كل أمر بالمشاريع المستهدفة.
- سجل حالات التنفيذ لكل مشروع.

مع صلاحيات:
- المدير فقط يرى وينشئ ويعدل الأوامر.
- لا وصول عام لهذه البيانات.

### 6. واجهة جديدة داخل الإدارة
إضافة صفحة جديدة مثل:
```text
/admin/project-commands
```
وتظهر في القائمة باسم: “أوامر التطوير”.

الصفحة تحتوي:
- زر “أمر جديد”.
- اختيار المشاريع المستهدفة.
- حقل الأمر.
- حالة الأمر.
- قائمة الأوامر السابقة.
- زر نسخ أمر المشروع.

## التحقق بعد التنفيذ
- التأكد أن الدخول لم يتغير.
- التأكد أن أزرار الحذف النهائي لم تعد تسبب حذفاً مباشراً.
- التأكد أن المدير يستطيع إنشاء أمر تطوير وتحديد المشاريع.
- التأكد أن البيانات محفوظة وتظهر بعد تحديث الصفحة.
- التأكد أن الصفحة واضحة وسريعة داخل لوحة الإدارة.

## النتيجة
ستصبح اللوحة مركز قيادة لمشاريعك: تراقب، تنظم، وتصدر أوامر تطوير للمشاريع الأخرى بشكل واضح وآمن، بدون حذف أو تغيير غير موافق عليه.

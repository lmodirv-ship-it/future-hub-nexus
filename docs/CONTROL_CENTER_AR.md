# 🎛️ مركز التحكم (Control Center)

لوحة مركزية لإدارة كل مواقعك من مكان واحد، **بدون أي تأثير على المواقع المنشورة** أو على دوميناتها.

## ما الذي يقدّمه؟

| الميزة | الوصف |
|---|---|
| 📋 قائمة المواقع | عرض كل مواقعك (الاسم، الدومين، الـ repo، السيرفر) |
| 🟢 مراقبة Uptime | فحص آلي كل **10 دقائق** لكل موقع مفعّل |
| 🔴 تنبيهات تلقائية | تسجيل تنبيه في صفحة "التنبيهات" عند تعطّل أي موقع |
| 🔄 مزامنة المرايا | جلب آخر كود من GitHub إلى سيرفر المرايا (اختياري) |
| 📊 سجلات | تاريخ الفحوصات لكل موقع |
| ⚙️ إدارة | إضافة، تفعيل/تعطيل، حذف |

## كيف أصل إليه؟

1. سجّل الدخول كمدير من `/login`
2. اذهب إلى `/admin`
3. من القائمة الجانبية: **مركز التحكم**

## إضافة موقع جديد

1. اضغط **"إضافة موقع"**
2. املأ:
   - **الاسم** (إجباري) — مثل: غسيل السيارات
   - **الدومين** (إجباري) — مثل: `example.com`
   - **رابط GitHub Repo** (اختياري) — لتفعيل المزامنة
   - **الفرع** — افتراضي `main`
   - **السيرفر الأصلي** — وصف نصّي (LWS, Vercel, ...)
   - **ملاحظات** (اختياري)
3. اضغط **"إضافة الموقع"**

الموقع سيُفحص في الدورة التالية (كل 10 دقائق) أو فوراً بضغط **"فحص الكل"**.

## ضمانات السلامة

✅ لا تغيير على DNS لأي دومين  
✅ لا تغيير على السيرفرات الأصلية للمواقع  
✅ المراقبة عبر HTTP GET فقط (read-only)  
✅ المزامنة (إن استُخدمت) تُجرى على سيرفر المرايا فقط — لا تمسّ السيرفر الأصلي

## الفحص الدوري

يعمل عبر **pg_cron** كل 10 دقائق ويستدعي:

```
POST https://future-hub-nexus.lovable.app/api/public/control/health-check
```

النتائج تُسجَّل في:
- `managed_sites` (الحالة الأخيرة)
- `site_health_history` (السجل الكامل)
- `alerts` (تلقائياً عند التعطل)
